## Tool naming
Use clear business verbs and nouns.

## Descriptions
Document purpose and boundaries in one sentence.

## Authentication
Use auth required for customer-sensitive operations.

## Permissions
Declare minimum required permission set.

## Error handling
Return deterministic, customer-safe errors.

## Performance
Keep handlers fast and idempotent when possible.

## Versioning
Track SDK and protocol versions in release notes and rollout plans.
