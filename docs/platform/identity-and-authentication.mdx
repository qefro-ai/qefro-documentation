# Identity Resolution & Authentication

Qefro is **not** an Identity Provider for customers.

| Concern | Question | Owner |
| --- | --- | --- |
| Identity Resolution | Who is this customer? | Qefro (`IdentityResolver`) |
| Customer authentication | May they perform this action? | **Organization** Customer Identity Service |
| Business Tool execution | Call the API with credentials | Qefro runtime (injection only) |

## Architecture

```text
Channel
  ↓
Identity Resolver
  ↓
Resolved Identity
  ↓
Organization Customer Identity Service
  ↓
Authentication Result
  ↓
Authentication Context (encrypted)
  ↓
Business Tool Runtime
  ↓
Business Tool
```

## What Qefro never does

- Send OTP
- Verify OTP
- Store OTP
- Authenticate customers
- Manage customer passwords
- Issue customer sessions or JWTs

## What the organization owns

- Customer lookup
- Whether authentication is required
- OTP / login / OAuth / passkeys
- Token issuance, refresh, and revocation

Configure under **Organization → Customer Identity Service** in the Admin Console.

## Identity API

Qefro calls:

```http
POST /identity
```

```json
{
  "organization_id": "org_001",
  "conversation_id": "conv_123",
  "channel": "whatsapp",
  "business_action": "download_invoice",
  "identity": {
    "type": "phone",
    "identifier": "+919876543210",
    "verified_by_channel": true
  }
}
```

Qefro does **not** send OTP, passwords, or sessions in this request.

### Responses

| Status | Runtime behavior |
| --- | --- |
| `authenticated` | Store Authentication Context; execute Business Tool |
| `challenge_required` | Relay the organization's message (OTP / login URL); pause |
| `denied` | Return error |
| `customer_not_found` | Return error |

When the customer replies to a challenge, Qefro **forwards** the reply to the Identity API. The organization verifies it.

## Business Actions

Configure only:

- `public`
- `verified_channel`
- `organization_challenge`

No provider configuration on Business Actions or Business Tools.

## Security

<Warning>
Authentication Context credentials are encrypted at rest, never logged, and never exposed to the AI model. OTP values and access tokens must never appear in Qefro logs.
</Warning>

<InfoBox>
Identity Resolution answers “who?”. The organization answers “may they?”. Business Tools only consume the context the runtime injects.
</InfoBox>
